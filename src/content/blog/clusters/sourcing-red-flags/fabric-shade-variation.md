### Why this happens even with a "correct" fabric order

Dyeing is a batch process — even ordering the exact same color code doesn't guarantee identical shade across different dye lots, especially if a reorder pulls from a different production run than your original sample. Without a specific lab-dip approval process and shade-matching check at the fabric stage, this variation surfaces for the first time when someone actually looks at a finished fit sample next to the original approved color reference — later than it should.

### Why it's a bigger problem across a large order

A shade variance that's barely noticeable on one garment becomes obvious across a rack of the same style in different colorways, or across a reorder next to units from the original run — exactly the scenario a growing brand runs into as it scales past a first small order.

### What actually catches this earlier

A lab-dip approval step before bulk dyeing, checked against a physical color reference (not just a Pantone code on paper), and a shade-check specifically at the fabric stage before cutting begins — not only at the finished-garment inspection stage, by which point an entire batch may already be affected.
